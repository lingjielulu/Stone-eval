# 短篇小说 F–T–P 人工标注台

这是一个只依赖 Python 标准库的本地双语标注工具。它用于人工标注：

- `F`（Foreshadow）：伏笔首次稳定出现的位置；
- `T`（Trigger）：伏笔进入兑现路径的触发事件；
- `P`（Payoff）：伏笔被解释、兑现、反转、否定或主题性回收的位置；
- 伏笔承载类型、叙事功能、兑现类型、原因解释和置信度。

## 启动

在仓库根目录运行：

```bash
python foreshadow/short_stories/annotation_app/server.py
```

浏览器打开 <http://127.0.0.1:8765>。

## 操作

1. 从顶部切换小说；有译文的作品可以切换英文、中文或中英对照。
2. 在正文中直接拖选任意文字，可以从段落中间开始并跨越多个段落。
   如果确实要选择整段，可以点击左侧段落编号。
3. 点击“设为 F / T / P”，或在焦点不处于输入框时按 `F`、`T`、`P`。
4. 填写三部分概括、分类与原因解释。
5. 草稿可以不完整；标记为“已复核”或“已裁定”时，F/T/P 范围、伏笔类型、叙事功能和原因解释为必填。
6. 使用 `Ctrl+S` / `Cmd+S` 保存。

CFPG 候选显示在左下角。导入只会创建待人工校正的草稿，不会自动成为金标。
点击正文上方的“显示 AI 结果”，可以把已有 CFPG 候选以 AI F/T/P 图层叠加
到正文；点击候选卡片的“正文定位”可只查看该条结果。再次点击按钮即可隐藏，
显示和隐藏操作不会修改人工标注文件。

中英文段落边界不一定完全对应（译者注等内容会造成错位），所以每个
F/T/P 都分别保存 `selection_en` 和 `selection_zh`，其中包括起止段落、
段内字符位置和精确摘录。对照视图中每次只选择一种语言；中文和英文范围
需要分别拖选、分别赋值。

## 数据位置

人工标注按小说分别保存到：

```text
foreshadow/short_stories/dataset/annotations/manual/{story_id}.json
```

这里与已有 YAML seed gold、弱规则候选和 CFPG 实验结果分开，避免覆盖历史数据。服务端使用临时文件加原子替换写入。

中文译文目前只有 8 篇，且授权状态尚待核验；界面会禁用不存在的语言版本。具体来源审计见 `dataset/docs/chinese_translation_audit.json`。
