# 数据集选篇与阶段建设设计

本文档记录 `foreshadow_causality_benchmark` 的选篇理由、候选池、批次规划和当前 seed 样本定位。目标不是按文学史地位排序，而是为伏笔系统和因果系统提供**结构信号强、可标注、可验证、可扩展**的短篇/中篇小说样本。

## 项目定位

这个 benchmark 服务后续长篇小说续写 Story Engine 的前期验证。长篇文本中需要持续追踪人物、事件、伏笔、因果链和主题线索，但直接从《红楼梦》《战争与和平》这类超长篇开始会带来三个问题：

- 文本规模大，人工标注和一致性校验成本高。
- 文化知识、隐喻和历史语境依赖重，容易把系统能力评测和外部知识评测混在一起。
- 伏笔和因果链跨越极长距离，早期系统很难定位错误来源。

因此第一阶段使用结构封闭、长度较短、因果和伏笔较明确的中短篇小说，先验证抽取、标注、校验和评测流程。

## 选篇原则

优先选择：

- **短篇/中篇、结构封闭**：文本边界清楚，事件链可完整读完和复核。
- **结构信号强**：线索、规则、物件、心理异常、空间约束或社会机制在后文有明确功能。
- **可标注性高**：事件节点、伏笔节点、触发事件和因果边可以被不同标注者较稳定地复现。
- **因果关系清楚**：能区分时间顺序与真实因果、动机因果、物理因果、心理因果或社会制度因果。
- **伏笔回收明确**：早期信息能对应后文 payoff、误导、反转、失败或主题性回收。
- **合法可获取**：优先公版或开放授权文本，便于复现、引用和后续发布。

第一阶段暂不优先选择：

- 超长篇小说。
- 强意识流文本。
- 过度依赖隐喻和开放解释的文本。
- 弱情节、强状态文本。
- 需要大量外部文化注释才能判断伏笔功能的文本。

## 优先测试类型

第一批样本主要覆盖三大类能力，并保留自然主义和中文迁移样本：

- **推理/谜题型**：线索、红鲱鱼、真相回收清楚，适合测试 clue/payoff matching。
- **反转/寓言型**：早期信息在结尾被重新解释，适合测试 retrospective foreshadowing。
- **心理/命运因果型**：人物心理、社会处境、制度压力或自然规则驱动事件发展，适合测试 event causality。
- **自然主义短篇**：物理规则和环境约束明确，适合测试 rule-based causality。
- **中文现代/古典短篇**：为后续中文长篇续写项目迁移预留样本。

## 核心候选内容简介

- **《斑点带子案》**：福尔摩斯调查一桩密室式死亡案。异常声响、通风口、假铃绳、固定床、牛奶盘和保险柜等线索最后共同解释为一套杀人机关。适合区分有效伏笔、红鲱鱼和解释性回收。
- **《红发会》**：一个红发男子被奇怪组织雇去抄百科，背后其实是犯罪团伙调虎离山，以便在其店铺附近挖地道抢银行。表面荒诞事件最终都能解释为犯罪计划的功能性因果。
- **《项链》**：女主人公借项链参加舞会，丢失后倾家荡产赔偿，多年后才发现原项链是假的。核心是物件误认、虚荣动机、阶层欲望和长延迟反转。
- **《麦琪的礼物》**：一对贫穷夫妻各自牺牲最珍贵的东西给对方买礼物：妻子卖头发买表链，丈夫卖怀表买发梳。两条因果线互相失效，却共同回收为爱的证明。
- **《泄密的心》**：叙述者杀人后藏尸，却因幻听死者心跳而崩溃自首。它的伏笔不是物件型，而是心理症状型：对“眼睛”“声音”“理智”的反复强调预告精神失控。
- **《一桶白葡萄酒》**：复仇者用 Amontillado 作为诱饵，利用对方虚荣心将其一步步引入地下墓穴并活埋。适合测试复仇动机、欺骗行为链和空间不可逆。
- **《生火》**：男子在极寒荒野中独行，低估自然规则，掉入冰水后必须生火，火被雪扑灭，手指冻僵后无法再点火，最终死亡。它适合作为非社会因果/物理因果测试集。
- **《最后一片叶子》**：病中的琼西把窗外藤叶的存亡和自己的生命绑定，苏和老画家贝尔曼的行动逐步把“叶子”从自然物转化为心理支撑。结尾揭示最后一片叶子其实是贝尔曼冒雨画在墙上的“杰作”，形成象征伏笔、信念因果和牺牲型回收。
- **《药》**：华老栓买人血馒头给儿子治病，另一边革命者夏瑜被处死。两条线在坟场和花环处形成象征性回收，核心是民众愚昧、革命孤独和主题层 payoff。
- **《促织》**：皇帝好斗蟋蟀导致地方摊派，成名一家被官僚压力推入绝境，儿子魂化促织后完成奇幻回收并反转家庭命运。适合测试权力层级因果。

## 分阶段建设

### Phase 1: 强结构、低歧义 MVP

优先用于打通下载、清洗、段落定位、候选生成、人工标注和校验流程：

1. 《斑点带子案》
2. 《项链》
3. 《麦琪的礼物》
4. 《生火》
5. 《泄密的心》

这些作品的伏笔、触发事件、回收和主因果链相对清晰，适合建立 seed gold 和标注一致性基线。若资源有限，最小启动集可以先做三篇：《斑点带子案》《项链》《生火》。它们分别代表推理线索、物件反转和物理因果，结构差异足够大，但标注难度都不高。

### Phase 2: 强结构、中等歧义扩展

加入更多象征、社会机制、心理因果和中文迁移样本：

1. 《一桶白葡萄酒》
2. 《最后一片叶子》
3. 《红发会》
4. 《促织》
5. 《药》

这一阶段用于测试系统能否处理象征性伏笔、社会机制因果、心理因果和奇幻回收，而不只处理物理线索。

### Phase 3: 复杂结构与不确定因果

加入多版本叙事、叙事层冲突、社会制度网络和弱情节压力测试：

1. 《竹林中》：多叙述者、多版本因果、真相不确定。
2. 《鹰溪桥上》：叙述层因果与真实世界因果冲突。
3. 《祝福》：社会制度因果网络。
4. 《在流放地》：装置伏笔、制度因果、暴力机器反噬。
5. 契诃夫《第六病室》或《带小狗的女人》：弱情节、强状态压力测试。

## 标注任务设计

建议不要只标“伏笔/回收”，而是拆成更稳定的结构。

### Foreshadowing-Trigger-Payoff 三元组

```text
Foreshadowing: 早期出现的线索/物件/话语/异常/规则
Trigger: 使伏笔进入兑现路径的事件
Payoff: 后期产生的可观察结果
```

例：《斑点带子案》

```text
Foreshadowing: 房间里有通风口和假铃绳
Trigger: 福尔摩斯夜间守在房间中，观察蛇的行动
Payoff: 发现蛇通过通风口和铃绳进入受害者床边
```

### Event Causal Edge

每个事件对之间标注关系类型：

```text
A causes B
A enables B
A motivates B
A prevents B
A reveals B
A misleads reader about B
A symbolically echoes B
```

例：《生火》

```text
A: 主角脚湿
B: 主角必须立刻生火
Relation: causes / necessitates
```

## 伏笔类型 Taxonomy

| 类型 | 定义 | 典型作品 |
|---|---|---|
| `object` | 物件早期出现，后期发挥关键功能 | 《项链》《杜十娘怒沉百宝箱》 |
| `rule` | 早期建立规则，后期按规则兑现 | 《生火》《促织》 |
| `psychological` | 人物心理异常预示后续行为 | 《泄密的心》 |
| `spatial` | 空间结构预示后续危险 | 《一桶白葡萄酒》《斑点带子案》 |
| `social` | 社会习俗/制度预示暴力或悲剧 | 《药》《祝福》 |
| `symbolic` | 意象在后文形成主题性回收 | 《药》《最后一片叶子》 |
| `red_herring` | 误导读者的伪线索 | 《斑点带子案》 |
| `retrospective` | 读完结尾后才意识到前文是伏笔 | 《鹰溪桥上》《最后一片叶子》 |

## 与标注 schema 的关系

选篇设计直接对应以下标注能力：

- 推理和机关样本：验证 `red_herring`, `spatial`, `object`, `reveals`, `misleads_about`。
- 反转样本：验证 `retrospective`, `delayed_revelation`, `motivates`, `reveals`。
- 自然主义样本：验证 `rule`, `causes`, `prevents`, `strength: strong`。
- 复仇/犯罪样本：验证 `spatial`, `motivates`, `enables`, `causes`, `irreversible`。
- 社会寓言/中文样本：验证 `social`, `symbolic`, `symbolically_echoes`。
- 多版本叙事样本：验证 `certainty` 和 `narrative_reality_level` 的 `disputed`, `symbolic`, `dream_or_hallucination`。

这样设计可以让每次新增作品都明确回答一个问题：它为伏笔系统或因果系统增加了哪一种可测能力？

## 不建议第一批使用的作品类型

不是作品不好，而是不适合验证系统初版：

| 作品类型 | 原因 |
|---|---|
| 契诃夫很多短篇 | 弱情节、强状态，伏笔不总是兑现。 |
| 卡佛短篇 | 极简主义，事件因果常被压低。 |
| 博尔赫斯部分短篇 | 概念结构强，但事件因果未必常规。 |
| 《红楼梦》章节 | 伏笔系统极强，但规模太大、文化知识依赖太重。 |
| 意识流小说 | 因果关系大量内化，标注一致性差。 |

等系统能稳定处理《项链》《斑点带子案》《生火》之后，再用契诃夫、卡佛、鲁迅《孤独者》这类弱因果作品做压力测试。

## 诗词意象堆叠扩展（Poetry Imagery Stacking Extension）

### 设计动机

古典诗词中大量使用「意象堆疊」手法——多個名詞意象並置，不使用動詞或連詞連接，純以物象的累積與對比產生情感效果。這種手法是測試伏筆系統中 **`symbolic` foreshadowing** 和 **意象級事件提取** 的理想材料：

- 文本極短（4–52 行），人工標注成本低，便於快速迭代標注規範。
- 意象堆疊是「靜態事件」——每個名詞組合本身就是一個象徵性事件節點，不依賴敘事時間流。
- 意象之間的關係是非因果但高度結構化的（並列、對比、遞進、轉折），非常適合訓練系統識別 **非敘事性伏筆結構**。
- 詩詞中的伏筆通常不指向詩內 payoff，而是指向詩外的情感或命運——這迫使系統處理更抽象的因果類型。

### 選篇原則

- **意象堆疊密度高**：每句包含三個或以上名詞意象並置（如「枯藤老樹昏鴉」「雞聲茅店月」）。
- **結構清晰**：意象堆疊 → 情感收束的 pipeline 明確可辨。
- **文本完整**：選取的是全篇（而非摘句），保留完整的意象→情感結構。
- **公有領域**：全部為清代及以前作品，無版權爭議。

### 標注策略差異

| 維度 | 小說標注 | 詩詞標注 |
| --- | --- | --- |
| `event_type` | 以 `action`/`dialogue`/`decision` 為主 | 以 `symbolic`/`internal_state`/`revelation` 為主 |
| `causal_edge` 關係 | `causes`/`enables`/`motivates` 為主 | `symbolically_echoes`/`reveals` 為主 |
| `foreshadowing_type` | `object`/`rule`/`psychological` 為主 | 幾乎全是 `symbolic` |
| `payoff_type` | `literal`/`delayed_revelation`/`misdirection` | 以 `symbolic` 和 `delayed_revelation` 為主 |
| 人物狀態 | 按事件分段記錄動機/知識/目標 | 多為抒情主體的穩定情感狀態 |
| `narrative_reality_level` | `real` 為主，少量 `narrated` | 絕大多數為 `symbolic` |

### 紅樓夢第五回的伏筆系統特色

第五回的特殊性在於它是**元伏筆（meta-foreshadowing）**——詩詞不僅包含意象堆疊，每個堆疊本身就是對整部小說人物命運的預言。標注時需注意：

- **雙層封裝**：詩內意象之間有因果/呼應關係（第一層伏筆），詩與小說人物命運之間有預言關係（第二層伏筆）。本標注主要處理第一層。
- **姓名嵌入**：玉帶林→林黛玉、金簪雪→薛寶釵、湘江水逝楚雲飛→史湘雲——意象選擇服務於姓名雙關，增加堆疊的語義密度。
- **物件鏈**：葬花吟中的花鋤→錦囊→淨土構成一個完整的儀式工具鏈，每個物件都是下一步行動的伏筆。
- **收尾閉環**：從對聯（入門）→判詞（閱冊）→曲（聆樂）→賦（描繪仙境），構成一個空間與體裁遞進的完整閉環。

### 與整體 Benchmark 的關係

詩詞意象堆疊擴展為伏筆系統增加了以下可測能力：

- **純意象級事件提取**：小說中事件多為行動/對話，詩詞中事件為純物象，測試系統能否將「枯藤」識別為一個事件節點。
- **非時間性 causal edge**：小說中的因果邊多為時間序列，詩詞中的邊為空間性/對比性（A 與 B 並置，A 反襯 B）。
- **最短文本伏筆**：天淨沙·秋思全文僅 28 字，但包含 12 個意象和一個完整的伏筆結構。
- **元伏筆結構**：紅樓夢判詞/曲是伏筆的伏筆——詩內意象堆疊的 payoff 不在詩內而在小說全篇，測試系統能否處理跨文本層級的伏筆。
