# 数据集选篇与阶段建设设计

本文档记录 `foreshadow_causality_benchmark` 的选篇理由、批次规划和当前 seed 样本定位。目标不是按文学史地位排序，而是为伏笔系统和因果系统提供结构信号强、可标注、可验证、可扩展的短篇小说样本。

## 项目定位

这个 benchmark 服务后续长篇小说续写 Story Engine 的前期验证。长篇文本中需要持续追踪人物、事件、伏笔、因果链和主题线索，但直接从《红楼梦》《战争与和平》这类超长篇开始会带来三个问题：

- 文本规模大，人工标注和一致性校验成本高。
- 文化知识、隐喻和历史语境依赖重，容易把系统能力评测和外部知识评测混在一起。
- 伏笔和因果链跨越极长距离，早期系统很难定位错误来源。

因此第一阶段使用结构封闭、长度较短、因果和伏笔较明确的中短篇小说，先验证抽取、标注、校验和评测流程。

## 选篇原则

优先选择：

- **结构信号强**：线索、规则、物件、心理异常或社会机制在后文有明确功能。
- **可标注性高**：事件节点和因果边可以被不同标注者较稳定地复现。
- **因果关系清楚**：能区分时间顺序与真实因果、动机因果、物理因果或社会制度因果。
- **伏笔回收明显**：早期信息能对应后文 payoff、误导、反转或主题性回收。
- **合法可获取**：优先公版或开放授权文本，便于复现、引用和后续发布。

第一阶段暂不优先选择：

- 超长篇小说。
- 强意识流文本。
- 过度依赖隐喻和开放解释的文本。
- 弱情节、强状态文本。
- 需要大量外部文化注释才能判断伏笔功能的文本。

## 优先类型

第一批样本主要覆盖以下类型：

- **推理短篇**：线索、红鲱鱼、真相回收清楚，适合测试 clue/payoff matching。
- **反转短篇**：早期信息在结尾被重新解释，适合测试 retrospective foreshadowing。
- **自然主义短篇**：物理规则和环境约束明确，适合测试 event causality。
- **复仇/犯罪短篇**：人物动机、空间推进和行为链清楚。
- **中文现代/古典短篇**：为后续中文长篇续写项目迁移预留样本。

## 分阶段建设

### Phase 1: 强结构、低歧义 MVP

优先用于打通下载、清洗、段落定位、候选生成、人工标注和校验流程：

1. 《斑点带子案》
2. 《项链》
3. 《麦琪的礼物》
4. 《生火》
5. 《泄密的心》

这些作品的伏笔、触发事件、回收和主因果链相对清晰，适合建立 seed gold 和标注一致性基线。

### Phase 2: 强结构、中等歧义扩展

加入更多象征、社会制度和心理因果：

1. 《一桶白葡萄酒》
2. 《摸彩》
3. 《最后一片叶子》
4. 《促织》
5. 《药》

这一阶段用于测试系统能否处理象征性伏笔、社会机制因果、心理因果和奇幻回收，而不只处理物理线索。

### Phase 3: 复杂结构与不确定因果

加入多版本叙事、叙事层冲突和弱情节压力测试：

1. 《竹林中》：多叙述者、多版本因果、真相不确定。
2. 《鹰溪桥上》：叙述层因果与真实世界因果冲突。
3. 《祝福》：社会制度因果网络。
4. 《在流放地》：装置伏笔、制度因果、暴力机器反噬。
5. 契诃夫《第六病室》或《带小狗的女人》：弱情节、强状态压力测试。

## 当前 seed 样本定位

当前已落地五篇：

- **《斑点带子案》**：推理/机关型样本，用于测试物理线索、红鲱鱼和侦探式因果重建。
- **《项链》**：反转现实主义样本，用于测试物件伏笔、人物虚荣动机、长延迟 payoff。
- **《生火》**：自然主义样本，用于测试环境规则、物理因果链、规则违背导致的不可逆失败。
- **《药》**：中文现代象征样本，用于测试双线结构、社会误解、人血馒头的负向 payoff 和花环的象征性回收。
- **《促织》**：中文古典奇幻样本，用于测试权力层级因果、官僚压力传导、魂化促织的 delayed revelation 和命运逆转。

这五篇覆盖了五种互补结构：推理线索链、反转物件链、自然物理链、中文现代象征链和中文古典制度/奇幻链。它们适合作为最小可用 seed，不代表完整第一阶段。后续应继续补齐《麦琪的礼物》和《泄密的心》，并可继续扩展《红发会》《一桶白葡萄酒》等强结构样本。

## 与标注 schema 的关系

选篇设计直接对应以下标注能力：

- 推理和机关样本：验证 `red_herring`, `spatial`, `object`, `reveals`, `misleads_about`。
- 反转样本：验证 `retrospective`, `delayed_revelation`, `motivates`, `reveals`。
- 自然主义样本：验证 `rule`, `causes`, `prevents`, `strength: strong`。
- 社会寓言/中文样本：后续验证 `social`, `symbolic`, `symbolically_echoes`。
- 多版本叙事样本：后续验证 `certainty` 和 `narrative_reality_level` 的 `disputed`, `symbolic`, `dream_or_hallucination`。

这样设计可以让每次新增作品都明确回答一个问题：它为伏笔系统或因果系统增加了哪一种可测能力？
